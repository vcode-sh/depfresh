# depfresh 2.0.2 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the reviewed Plan 030 hotfix as `depfresh@2.0.2` and prove the exact public npm
and GitHub artifact before beginning the `2.1.0` migration.

**Architecture:** Prepare one version-coupled release candidate, replay the repository's exact
local artifact gates, require hosted `main` success for the candidate SHA, then let an immutable
`v2.0.2` tag trigger the existing OIDC release workflow. Public completion requires npm, GitHub,
the workflow artifact, and provenance to agree.

**Tech Stack:** TypeScript, Node `24.15.0`, pnpm `10.33.0`, npm `11.12.x`, Vitest, Biome, GitHub
Actions, npm trusted publishing, GitHub Releases.

## Global Constraints

- All code, documentation, plans, and commit messages are English.
- Preserve `docs/releases/v2.0.0.md`, `docs/releases/v2.0.1.md`, published schema bytes, and the
  immutable `v2.0.0`/`v2.0.1` tags.
- Keep Node `>=24.15.0`, pnpm `10.33.0`, ESM-only output, and Plan 030 behavior.
- Use disposable HOME, cache, store, install, and artifact directories for every proof.
- Do not publish locally, create a floating `v2` tag, force-move `v2.0.2`, bypass hosted OIDC, or
  claim public completion from local evidence.
- Stop before tag creation unless hosted `main` is green for the exact release commit.

## Drift Check and Stop Conditions

Before editing, run `git status --short`, verify Plan 030 is recorded complete, and recheck live npm
and GitHub absence for `2.0.2`/`v2.0.2`. Stop if the package or tag already exists unexpectedly,
Plan 030 evidence is incomplete, release surfaces overlap unrelated work, any local/hosted gate is
red, artifact identities disagree, or publication would require moving a public tag.
