# depfresh 2.1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the reviewed command-level apply and Visual+ v2 product as `depfresh@2.1.0` and
prove the exact public artifact and terminal journeys.

**Architecture:** Prepare one version-coupled minor-release candidate after Plans 032-035 pass,
replay source and packed CLI gates including real PTY journeys, require hosted `main` success,
then publish only through the immutable `v2.1.0` OIDC tag workflow. Public proof reruns success
and safety journeys against installed npm bytes.

**Tech Stack:** TypeScript, Node `24.15.0`, pnpm `10.33.0`, npm `11.12.x`, Vitest, Biome, GitHub
Actions, real PTY tests, npm trusted publishing, GitHub Releases.

## Global Constraints

- All code, documentation, plans, and commit messages are English.
- Begin only after Plans 032-035 are complete with independent approvals.
- Preserve all historical release notes, published schema bytes, and immutable release tags.
- Keep Node `>=24.15.0`, pnpm `10.33.0`, ESM-only output, no new runtime/native dependency, and the
  approved Visual+ fallback matrix.
- Use disposable HOME, cache, store, install, repository, PTY, and artifact paths for every proof.
- Do not publish locally, create a floating `v2` tag, force-move `v2.1.0`, bypass hosted OIDC, or
  claim public completion from local/hosted evidence alone.
- Stop on any missing row/target, terminal leak, filesystem mismatch, schema drift, failed recovery
  assertion, ambiguous artifact identity, or public provenance mismatch.

## Drift Check and Stop Conditions

Before editing, run `git status --short`, verify Plans 032-035 are complete with recorded review and
PTY evidence, and recheck live npm/GitHub absence for `2.1.0`/`v2.1.0`. Stop if any prerequisite is
incomplete, release surfaces overlap unrelated work, local/hosted/public proofs disagree, a required
PTY gate is skipped, publication would require manual npm access or tag movement, or provenance
cannot bind the exact artifact to the exact release commit.
