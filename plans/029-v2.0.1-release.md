# depfresh 2.0.1 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this
> plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the reviewed Plan 028 product as `depfresh@2.0.1` and prove the exact public npm
and GitHub release artifact.

**Architecture:** Prepare one version-coupled release candidate on `main`, prove it locally and in
hosted CI, then let the immutable `v2.0.1` tag trigger the existing OIDC release workflow. Record
public evidence only after npm, GitHub, and provenance agree.

**Tech Stack:** TypeScript, pnpm `10.33.0`, Node `24.15.0`, npm `11.12.x`, Vitest, Biome, GitHub
Actions, npm trusted publishing, GitHub Releases.

## Global Constraints

- All code, documentation, plans, and commit messages are English.
- Preserve `docs/releases/v2.0.0.md`, published v1 schema bytes, and the immutable `v2.0.0` tag.
- Keep Node `>=24.15.0`, pnpm `10.33.0`, ESM-only output, and all Plan 028 runtime contracts.
- Use disposable HOME/cache/store/install/artifact directories for release proof.
- Do not publish locally, create a floating `v2` tag, force-move `v2.0.1`, or bypass hosted OIDC.
