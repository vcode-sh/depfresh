# depfresh 2.1.2 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve every approved audit finding and publish an exact, independently verified
depfresh 2.1.2 release.

**Architecture:** Keep existing public contracts and fail-closed execution boundaries intact. Add
small tested helpers at release, cache, error, GitHub, HTTP, test-topology, and validation-index
boundaries, then prove the exact packed artifact before any irreversible release action.

**Tech Stack:** TypeScript 7, ESM, Node 24.15.0, pnpm 11.15.1, Vitest 4, Biome 2, GitHub Actions,
npm 12.0.1 release tooling.

## Global Constraints

- All code, documentation, workflow text, release notes, and commits are in English.
- Node is locked to exactly 24.15.0; `@types/node` stays on major 24.
- Use current compatible versions for every other direct dependency and action.
- No new runtime dependency is permitted for this hardening release.
- Published v1 schema bytes and all CLI/library machine contracts remain unchanged.
- Unknown, unavailable, ambiguous, stale, or bounded-out evidence never becomes success.
- Secrets must never be logged, hashed into a persistent identity, or stored in cache keys.
- Every behavior change uses a witnessed RED then GREEN focused test.
- Preserve unrelated worktree changes; stop if the tree ceases to contain only owned changes.
- No push or tag occurs until all local release gates and final review pass.
- The npm publication must be the exact locally and hosted-verified tarball for tag `v2.1.2`.
