# depfresh 2.0.2: trustworthy legacy write receipts

depfresh 2.0.2 is a focused safety hotfix for normal `check --write` runs. It keeps the public
machine schemas and the existing 2.0.x package-by-package write model unchanged while making Git
preflight evidence and human receipts truthful on large or partially unavailable repositories.

## What changed

- exact-target Git evidence no longer inventories the complete tracked index;
- VCS preflight blocks retain VCS_UNAVAILABLE instead of WRITE_FAILED;
- repeated non-success writes are grouped by physical target;
- normal 2.0.x writes remain package-by-package until 2.1.0.

Exact Git failures remain attached to the canonical target they block, global-manager outcomes stay
outside local file counts, and the complete receipt is emitted as one ordered stdout block. Reverted
outcomes show recovery counts, while strict resolution and post-write failures retain the final
command exit instead of printing a contradictory success.

The local package verifier now applies fixed deadlines to its child processes and removes only the
temporary directories it owns. No published schema bytes, invocation-authority rules, manager
phases, artifact-trust boundaries, runtime requirement, or package-manager support changed.

## Upgrade

```bash
pnpm add -D --save-exact depfresh@2.0.2
pnpm exec depfresh --version
```

Node.js `24.15.0` or newer remains required.

## Release verification

The immutable `v2.0.2` tag workflow is configured to rebuild under exact Node `24.15.0`, install
isolated npm `11.12.0`, run schema, type, lint, coverage, smoke, demo, and installed-package gates,
publish only the verified tarball through npm trusted publishing, verify the public package, and
create the GitHub release from this file. It does not create or move a floating `v2` tag.

Before tag creation, the exact candidate must pass the complete local release workflow and the
hosted `main` workflow at the release commit. Public completion additionally requires npm, GitHub,
the retained workflow artifact, and provenance to agree on the same package bytes.
