# depfresh 2.0.0: inspect it, plan it, apply it, prove it

depfresh 2.0 turns dependency maintenance into an evidence-backed workflow. Inspect the physical
repository, review a deterministic immutable plan, grant only the authority needed for the chosen
operation, and evaluate what the filesystem and package managers actually did. Unknown state is
never reported as success.

## Highlights

- Deterministic schema-v1 `capabilities`, `inspect`, `plan`, `apply`, global-plan, and global-apply
  contracts with strict validators and packaged JSON Schemas.
- Exact dependency-occurrence identity across manifests, overrides, resolutions, pnpm/Bun/Yarn
  catalogs, workspaces, managers, lockfiles, declared runtimes, and target Git state.
- Ordered occurrence policy with complete selector and winning-rule traces. Configuration shapes
  policy but never grants side-effect authority.
- Stale-safe per-file apply with root-local locking, durable journal evidence, target revalidation,
  observed outcomes, and best-effort recovery without claiming repository-wide atomicity.
- Explicit npm, pnpm, and Bun lockfile/install phases with fixed no-shell argument arrays,
  lifecycle scripts disabled, exact manager evidence, and conservative process observation.
- Honest non-transactional global updates with manager preflight and fresh per-item inventory.
- Fingerprinted runtime, peer, cohort, release, maturity, deprecation, completeness, staleness, and
  passive trust-presence signals. Passive metadata is never presented as verified trust.
- Optional exact public-npm artifact verification for npm 11.12.x. Final lockfile SHA-512 and
  installed identity are bound before isolated `npm audit signatures` execution; signature and
  provenance results remain independent.
- One packaged automation workflow: deterministic capability discovery, a public-API-only skill,
  fixed recipes/examples, and a version-coupled GitHub Action with semantic result validation.

## Start here

```bash
depfresh capabilities --json
depfresh inspect --json
depfresh plan --json > depfresh-plan.json
depfresh apply --json --write --plan-file depfresh-plan.json
```

Inspect and plan exit `1` for a valid document containing actionable, blocked, risky, unknown, or
otherwise incomplete findings; exit `2` is fatal. Apply exit `0` is observed `applied` or `noop`;
exit `1` retains a schema-valid conflict, recovery, failure, or unknown result.

## Breaking changes

- Node.js `>=24.15.0` is required. The cache uses built-in `node:sqlite`; no native cache addon is
  distributed.
- Exact manifest pins require explicit `--include-locked`; global installed versions remain
  observations rather than manifest pins.
- Repeated names in different physical declarations are independent occurrences.
- Apply requires a reviewed plan plus explicit invocation authority. Legacy shell-string
  post-write execution flags are rejected.
- Dirty, stale, escaped, aliased, hard-linked, ambiguous, or unobservable targets block instead of
  weakening preconditions.
- Direct global writes use the observed global state machine and never claim rollback.
- Automation should discover behavior with `depfresh capabilities --json` and validate complete
  result contracts instead of inferring behavior from help text.

## Deliberate limits

- Manager phases are supported on Linux and macOS for reviewed npm 10/11, pnpm 10/11, and Bun
  `>=1.2.0 <2.0.0`. Windows manager execution, Yarn execution, and legacy `bun.lockb` are
  unsupported.
- File replacement is atomic per file, not across a repository. Install trees, package-manager
  caches, and global updates are non-transactional effects.
- Exact artifact verification supports public npm artifacts with npm 11.12.x. Pnpm, Bun, JSR,
  private registries, project npm configuration, and unavailable/offline verification do not pass.
- npm does not expose trustworthy positive per-artifact signature coverage through its aggregate
  result, so signature verification never reports pass. Provenance passes only for one exact
  verified SLSA v1 subject and SHA-512 digest match.
