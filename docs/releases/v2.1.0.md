# depfresh 2.1.0: command-level safety and Visual+ journeys

depfresh 2.1.0 is the minor release for the reviewed command-level local apply lifecycle and
Visual+ terminal results. It keeps machine schemas, public library contracts, and explicit
invocation authority stable while making local write receipts and terminal fallback behavior more
useful and more honest.

## What changed

### Command-level preflight

- Command-level preflight covers every selected physical target before the first replacement.
- One root-local lock and journal lifecycle covers the prepared local command, while unrelated dirty
  targets remain outside the selected write scope.

### Visual+ hierarchy

- Visual+ hierarchy keeps topology, severity, major risk, owner impact, shared dependencies, and final receipts legible.
- Complete, safety-block, and recovery-first journeys retain exact operation counts, physical paths,
  lifecycle evidence, and conservative next actions before their final exit status.

### Complete fallbacks

- Complete fallbacks preserve append-only output in pipes, CI, and TERM=dumb without terminal control leakage.
- NO_COLOR and narrow capable terminals retain the eligible plain-terminal journey without changing
  JSON, interactive, global, veto-capable, or public library contracts.

### Atomicity and recovery

- Replacement is atomic per file only; a repository is not an atomic transaction, and recovery remains best effort.
- Incomplete or unobservable recovery remains unknown and exits non-successfully.

## Upgrade

```bash
pnpm add -D --save-exact depfresh@2.1.0
pnpm exec depfresh --version
```

Node.js `24.15.0` or newer remains required.
