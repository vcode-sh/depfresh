# depfresh 2.0.1: exact workspace and catalog exclusions

depfresh 2.0.1 adds precise one-run controls for monorepos without changing persistent policy or
write authority. You can exclude one modeled workspace, one physical catalog name, or repeat both
flags while retaining deterministic evidence of exactly what was excluded.
The exact flags are `--exclude-workspace` and `--exclude-catalog`.

## What is new

```bash
depfresh --exclude-workspace apps/admin
depfresh -w --exclude-catalog payments
depfresh plan --json \
  --exclude-workspace packages/legacy \
  --exclude-catalog=mobile,v2 > depfresh-plan.json
```

- `--exclude-workspace <path>` matches one exact repository-relative workspace, including `.` for
  the root. It excludes direct declarations and proven catalog-consumer occurrences owned by that
  workspace, but never freezes a shared physical catalog owner.
- `--exclude-catalog <name>` matches every proven physical catalog with that exact name and its
  linked consumers. Commas, spaces, punctuation, and Unicode remain literal rather than becoming
  patterns.
- Both flags are repeatable and preserve first-seen order. Missing, unsafe, ignored, unavailable,
  ambiguous, or unmatched targets fail before registry, cache, interaction, plan-operation, or
  write work.

Use `--exclude` for dependency-name filtering and `--ignore-paths` only when repository discovery
itself should omit a path. For persistent selection, keep using declarative policy rules.

## Observable and reviewable selection

Normal table and JSON checks emit a durable derived selection receipt. Current machine plans use
strict Plan v2 and fingerprint exact request, entity, occurrence, decision, and summary bindings.
Apply validates those bindings and the original fingerprint before executing only the reviewed
eligible operations; it never silently rebinds selection against a changed repository.

Current capabilities output uses schema v2 and advertises the exact check/plan command scope. The
published Plan v1 and Capabilities v1 schema bytes remain unchanged, reviewed Plan v1 documents
remain apply-compatible, and apply output remains v1.

## Ownership and safety guarantees

- Workspace exclusion never propagates into a shared catalog owner.
- Catalog exclusion never captures unresolved consumers or same-name direct declarations.
- Same-named physical catalogs are all selected by stable source identity, never enumeration order.
- Invocation selection adds no write, process, install, verification, network, global, Git, or
  publishing authority.
- `inspect`, `apply`, `capabilities`, `global`, and `global-all` reject the new flags.

## Upgrade

```bash
pnpm add -D --save-exact depfresh@2.0.1
pnpm exec depfresh --version
pnpm exec depfresh capabilities --json
```

Node.js `24.15.0` or newer remains required. No runtime dependency, package-manager support,
manager-execution, artifact-verification, or platform boundary changed from 2.0.0.
